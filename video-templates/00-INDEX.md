## Архитектурный YAML-template (single-source-of-truth)

```yaml
# Любой шаблон собирается из 6 переиспользуемых слоёв:
template:
  format_ref:    # из 90 форматов (file 01-04)
  style_ref:     # из 12 эстетик (file 06)
  camera_preset: # из 6 готовых или custom из 65 building-blocks (file 07)
  audio_profile: # из 12 music-профилей (file 08)
  copy_pack:     # hook + caption + CTA + UTM (file 09)
  product:       # категория + must-show кадры (file 11)

  params:        # клиентские поля
    product_url:
    brand_color_anchor:
    cta_text:

  quality_gates: # из QC-чек-листа (file 10)

  publishing:    # platform tech-spec auto-adapt (file 10)
```

| Слой | Стоимость |

## Workflow интеграция

```
Клиент → выбирает шаблон в UI (формат + стиль + категория)
       → вводит params (URL товара + brand color + CTA-текст)
       → загружает реальные ассеты (если категория требует — для платьев/обуви/свадебного)
       ↓
Pipeline → собирает YAML по template_id
        → генерирует prompt для Veo 3.1 / Seedance 2 / Kling 3
        → отправляет на API
        → получает video output
        ↓
QC pipeline → 15 автоматических проверок
           → если fail → re-generate (max 3 попытки)
           → если manual review нужен → flag operator
           ↓
Post-render → FFmpeg overlay text + CTA + audio mix
           → loudness normalization (-14 LUFS)
           → format adaptation per platform (9:16 / 3:4 / 1:1)
           ↓
Delivery → доставка в личный кабинет клиента / Telegram / WhatsApp
        → загрузка на платформу через API (опционально)
        → tracking через UTM
```
