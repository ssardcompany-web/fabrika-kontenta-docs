## 1. Контекст и стратегическая установка

| Параметр | Решение |
|---|---|
| **Продукт** | Фабрика Контента — AI-генерация продающих видео для брендов одежды (короткие ролики 8–60 сек для Reels / Shorts / TikTok / карточек маркетплейсов) |
| **Технологический стек** | Veo 3.1, Sora 2, Nano Banana (плюс Kling / Higgsfield / Midjourney как fallback) |
| **Рынки** | **РФ + Аргентина** одновременно. Универсальный продукт, локализация — верхний слой |
| **Сегментов в работе** | 5 (см. раздел 2) — все обязательны, не приоритезируются на этапе концепции |
| **Стратегия offering-выбора** | A/B/C-тест на трафике: 5 single-target + 1 hub + 1 universal = 7 концепций × 2 языка = **14 лендингов** в split-тесте |
| **Pricing-модель** | **D — Гибрид:** под каждый сегмент своя ценовая модель и свой коридор (см. раздел 4) |
| **Конкурентная асимметрия** | РФ: AI-сервисов много (Neiro Card AI, SELLER MOON, агентства "под ключ" Связи / VideoHunter / Agentura), рынок учит сам себя. AR: локальных AI почти нет, глобалы без ARS-тарифов и Mercado Pago-биллинга — **зелёный океан** с инфраструктурными барьерами |
| **Регуляторное окно** | РФ: AI-видео ограничено маркетплейсами (≥30% реальных кадров на Ozon, бан лиц на WB) — конструируем под workaround. AR: свобода, можно pure-AI |
| **Подход к копи** | Двуязычно с нуля (не "сначала RU, потом AR"), носители RU + ES-AR пишут параллельно. **Voseo обязателен** в ES-копи ("vendé / creá / sumate"), не Spain-Spanish |

## 3. Value Proposition × сегмент × рынок

### 3.1 Общее ядро

> Превращаем один артикул одежды в десятки готовых продающих видео без съёмок, моделей, студии и логистики — за дни вместо месяцев, по цене UGC вместо продакшна.

**Три якорных value-driver, работающих на любом сегменте:**

| Driver | Что снимает | Чем измеряется |
|---|---|---|
| **Скорость** | Контент-голод (новинки еженедельно, обновление фото каждые 3–6 мес для WB-индекса актуальности) | Time-to-content: дни вместо недель |
| **Юнит-экономика** | Дорогие фотопакеты (24K+ ₽ за 10 артикулов; lookbook 100–250K ₽/день) | Cost-per-video: рубли/песо за единицу контента |
| **Масштабируемость** | Контент-разрыв ×200 (Lamoda 10K размещений/мес vs Ushatava 50); MFG 2716 SKU vs средний бренд утопает | Видео в месяц без потери качества |

## 4. Продукт-инвариант и Pricing-блок D (Гибрид)

### 4.1 Продукт-инвариант (одно ядро на всех)

| Слой | Что одинаково для всех сегментов |
|---|---|
| Технология | Veo 3.1, Sora 2, Nano Banana + Kling/Higgsfield/Midjourney fallback |
| Рабочий процесс | бриф → AI-генерация → доставка готового MP4 в нужный канал клиента |
| Метрики | time-to-content, cost-per-video, видео в месяц без потери качества |
| Гарантии | 12-месячная гарантия обслуживания + поддержка по часовому поясу клиента |

### 4.2 Pricing-карта (Гибрид по сегментам)

> **Цифры — ориентиры из ресёрча.** Окончательная калибровка — по результатам A/B/C-теста и первых пилотов. Опорные конкуренты в [pricing-deep-dive.md](../../research/_partials/08-pricing-deep-dive.md).

- **РФ-цены сегментов 3, 4** — ближе к рыночным аналогам (проектная цена коллекции, лидгена)

## 5. Локализация (RU + ES-AR с нуля)

Двуязычно с момента запуска. Не "сначала RU, потом AR". Носители RU и ES-AR пишут параллельно. **Voseo обязателен** в ES-копи ("vendé / creá / sumate"), не Spain-Spanish.

| Слой | RU | ES (AR) |

## 6. 7 лендингов: hero, оффер, воронка

### 6.1 Универсальные правила всех 7 лендингов (Google guideline applied)

| Правило | Что делаем |
|---|---|
| **3-second rule** | H1 + value proposition в первом экране без скролла |
| **Single primary CTA** | Один основной call-to-action на странице (single-CTA даёт 13.5% vs multi-CTA 10.5% по Unbounce) |
| **Mobile-first design** | Дизайн начинаем с экрана 375×812, потом масштабируем (79% визитов SaaS — мобильные) |

| **Message matching** | H1 = заголовок рекламного объявления, не вариация |
| **5–7 grade reading** | Простые слова, короткие предложения; никаких "интегрированных омниканальных решений" |

#### L7 — Universal pitch (one-page-fits-all)

**Hero RU:**
- H1: «Вы продаёте одежду. Мы делаем для неё видео.»
- Sub: «AI-генерация продающих роликов от 1 633 ₽/штука. Без съёмок, моделей и студии. Первый ролик за 24 часа — бесплатно.»
- CTA: [Получить первое видео бесплатно]

**Hero ES:**
- H1: «Vos vendés ropa. Nosotros le hacemos videos.»
- Sub: «Videos generados con IA desde USD 6.6/unidad. Sin sesión, sin modelos, sin estudio. Primer video en 24 horas — gratis.»
- CTA: [Recibí tu primer video gratis]

**Воронка:**
1. Hero с обещанием бесплатного первого видео
2. Universal pain: «Контент-голод. Дорогие фотосессии. Конкуренция китайцев»
3. Solution в 3 шага: загрузите фото → бриф → видео через 24 часа
4. Demo: видео-карусель из 8–10 роликов на разные товары (платье / куртка / кеды / детская)
5. Pricing-teaser: «От 1 633 ₽/ролик. Подписки, проекты или Enterprise — всё рассчитаем после первого тестового»
6. Trust: общие отзывы / цифры
7. Final CTA: форма 2 поля (контакт + ссылка на 1 артикул) — генерим тестовый ролик в 24 ч

**A/B переменные L7:** «бесплатно» в Hero vs цена в Hero; demo (грид vs карусель vs auto-play видео); CTA («бесплатное видео» vs «расчёт стоимости»)

<a id="sec-7"></a>
## 7. Протокол A/B/C-теста

### 7.1 Split трафика

| Источник трафика | На какие лендинги | Логика |
|---|---|---|
| Контекстная реклама на узкие ключи сегмента («фотосессия для wb») | L1-L5 (single-target соответствующего сегмента) | Узкое объявление — узкий лендинг (message matching) |
| Контекстная реклама на широкие ключи («контент для одежды») | L6 (hub) или L7 (universal) — split 50/50 | Тестируем какой формат держит широкий трафик |
| Холодный прямой трафик (organic, прямой URL) | L6 (hub) | Дефолтная точка входа |
| Outbound (LinkedIn DM) для Enterprise | L5 | Direct-pitch |
| Telegram-пост / WhatsApp-реклама в чатах селлеров | L1 | Узкая аудитория = узкий лендинг |

### 7.2 Метрики (по приоритету)

1. **Conversion Rate** (заявка / визит) — основная метрика победителя
2. **Cost per Lead (CPL)** — экономика канала
3. **Lead Quality Score** (ручная квалификация) — отсекаем мусор
4. **Cost per Qualified Lead (CPQL)** = CPL ÷ %quality
5. **Bounce Rate** — индикатор плохого message matching
6. **Time to first action** (скролл / клик / форма)
7. **Mobile vs Desktop CR** разрезом (79% мобильных)

### 7.3 Минимальная выборка для статзначимости

- При ожидаемом CR ≈ 4% (median SaaS Unbounce 3.8%) и MDE = 30%
- Минимум **N ≈ 1 200 визитов на каждый вариант** для p<0.05
- При 7 лендингов × 2 языка = **14 вариантов** → нужно **~16 800 визитов суммарно** для одного раунда теста
- Реально это **2–6 недель** при потоке 500–1 500 визитов/день

### 7.4 Длительность теста и критерий победителя

- Минимум **14 дней** в каждой языковой группе (учитываем weekly seasonality)
- Победитель = лендинг с CPQL-преимуществом ≥20% (статзначимо при p<0.05)
- Если 2+ лендинга в кластере не отличаются статзначимо → оставляем оба
- Если никто не выдаёт CR ≥ 3.8% (median) → меняем не лендинг, а ICP / месседж / канал

### 7.5 Раунды

- Раунд 1: тестируем все 7 концепций на одинаковом трафике, цель — найти топ-3
- Раунд 2: на топ-3 — A/B по pricing (цена в Hero) и по proof (логотипы / кейсы / цифры)
- Раунд 3: на топ-1 — A/B по форме (3 поля vs 5 полей) и по визуалам Hero

## 8. Что НЕ входит в концепцию (anti-scope)

Чтобы документ не размывался, явно фиксируем что **отсутствует и должно делаться отдельными артефактами**:

- **Маркетинговая стратегия** (каналы привлечения, бюджеты, контент-план, лид-магниты) — это Sub-project 2, отдельный документ
- **Реализация сайта** (HTML/CSS/JS, фреймворк, хостинг, CMS) — Sub-project 3
- **Платформа продукта** (workflow генерации, бэкенд, очередь Veo/Sora, доставка) — отдельный технический спек
- **Юр. документы** (договор оферты RU + cross-border USD-договор) — отдельный legal-спек
- **Финансовая модель** (P&L, unit-экономика, runway) — отдельный финансовый артефакт
- **Кейсы и сторителлинг бренда** — после первых клиентов
- **Расширение в Mexico/Chile/Colombia** — после стабилизации AR
- **Цифровые аватары / параметрические модели как отдельный продукт** — фаза 2 (есть в Enterprise-апгрейде)
- **Отдельная команда AR в Buenos Aires** — нет, виртуальный AM достаточно
